# TA531 STM32 控制板项目说明书

> 说明：本文根据当前固件 `main.c` 中体现的功能、外设初始化、CAN/LIN 协议处理和运行流程整理，用于项目交接、调试和后续维护。若实际原理图、BOM 或接插件定义与本文不一致，请以硬件设计文件为准，并同步更新本文档。

## 1. 项目背景与需求

### 1.1 项目背景

本项目基于 STM32G0 系列 MCU，面向 TA531 测试/执行控制场景，板卡需要同时承担以下任务：

- 控制触控/运动机构，实现 X/Y/Z 方向动作。
- 通过 CAN 接收上位机或整车测试系统命令，并反馈执行状态。
- 通过 LIN 与方向盘开关、网关或其他 LIN 节点通信，完成报文采集、转发和仿真。
- 采集 ADC/光照等传感器状态，用于环境检测和状态上报。
- 控制门锁/车门/继电器/KL15 等离散输出，支持车辆相关测试动作。
- 使用 OLED 提供现场调试信息、版本信息、状态提示和错误提示。
- 支持 Flash 参数保存，用于屏幕坐标标定、UID/密钥等配置管理。

### 1.2 主要需求

#### 运动控制需求

- 支持 X/Y 平面定位：
  - 电机 1、2 作为 Y 轴运动执行机构。
  - 电机 3 作为 X 轴运动执行机构。
  - 电机 4 预留/用于 Z 轴相关控制。
- 支持两类 RC 控制命令：
  - `0x064`：毫米/位置语义控制。
  - `0x065`：百分比语义控制。
- 支持屏幕坐标标定：保存 `X0/Y0` 与 `X1/Y1`，后续命令以屏幕标定范围进行映射。
- 支持同一 XY 目标下继续执行 `Z_code` 或 `XYMove` 的组合动作。
- 支持软限位与越界保护，避免机构移动到屏幕标定区域之外。
- 支持运动保护：卡滞、方向抖动、运动超时等异常时进入保护处理。

#### 通信需求

- CAN1 作为主要控制和状态通道：
  - 接收环境/门控/RC 运动控制报文。
  - 发送动作完成、XY 到位、XYMove 完成、门控 ACK 等反馈报文。
- CAN2 作为扩展/透传通道，用于电机或其他 CAN 通信需求。
- LIN1/2/3 支持多通道 LIN 通信，固件中包含 LIN 收包、RID 识别、调试计数和重启接收流程。

#### 人机交互与维护需求

- OLED 显示版本、ID 模式、运行状态、LIN/CAN 状态、错误信息。
- 方向按键支持手动移动机构：
  - `UP/DOWN` 控制 Y 轴。
  - `LEFT/RIGHT` 控制 X 轴。
  - `BUTTON` 用于确认/触发动作。
- 启动时支持 Flash 自检开关：
  - `CFG_BOOT_FLASH_VERIFY_RUN = 1`：执行 Flash 验证。
  - `CFG_BOOT_FLASH_VERIFY_RUN = 0`：跳过 Flash 验证，加快上电调试。

## 2. 板卡总体设计思路

### 2.1 总体架构

板卡采用“MCU 主控 + 多通信接口 + 电机/继电器执行 + OLED/按键调试”的总体架构：

```text
                ┌──────────────────────────────┐
                │          STM32G0 MCU          │
                │                              │
CAN1/CAN2  ────►│  CAN 命令解析 / 状态反馈       │────► 电机控制 CAN/驱动
LIN1/2/3   ◄───►│  LIN 收发 / RID 处理           │
ADC/光照   ────►│  传感器采集 / 状态判断         │────► OLED 调试显示
按键/拨码  ────►│  模式选择 / 手动控制 / 标定     │────► 继电器 / KL15 / 门控
Flash      ◄───►│  参数保存 / 屏幕坐标 / UID      │
                └──────────────────────────────┘
```

### 2.2 分层设计思路

固件逻辑可以分为以下几层：

1. **硬件抽象层**
   - STM32 HAL 初始化 GPIO、ADC、DMA、SPI、I2C、UART/LIN、FDCAN、TIM。
   - GPIO 控制继电器、KL15、LIN_EN、蜂鸣器、按键读取等。

2. **通信协议层**
   - CAN 报文接收回调中根据 ID 分发：环境状态、RC 运动控制、LIN 桥接、门控等。
   - LIN 回调中读取 UART 数据，识别常见 RID，并更新调试计数/数据状态。

3. **业务控制层**
   - RC 运动状态机：解析目标 XY、Z_code、XYMove、Reset。
   - 屏幕坐标映射：将逻辑坐标转换到标定后的机械坐标。
   - 门/继电器控制：状态变化时驱动输出并发送 ACK。

4. **保护与诊断层**
   - 运动保护：检测方向反复变化、位置停滞、超时。
   - 位置边界：通过 `Position_InBounds` 和 `Clamp_Position` 限制目标坐标。
   - OLED 状态页和错误页显示现场信息。

5. **参数管理层**
   - Flash 保存屏幕标定值、UID/密钥等信息。
   - 启动时可选择执行 Flash 自检或跳过自检。

### 2.3 模式选择思路

板卡通过 `IO_CFG_1` 至 `IO_CFG_4` 拨码/配置输入选择运行模式。固件注释中说明配置脚为上拉输入，拨码关闭时为高电平，拨码打开接地为低电平。

常见用途包括：

- `id1`：是否进入 MoC/机器人运动控制主流程。
- `id2`：与稳定性测试模式等组合条件相关。
- `id3`：决定 `XYMove` 是解释为目标值还是增量值。
- `id4`：百分比模式中选择 `/100` 或 `/255` 映射语义。

## 3. 硬件架构说明

### 3.1 主控 MCU

- 使用 STM32G0 系列 MCU。
- 主要外设：
  - FDCAN1 / FDCAN2
  - USART1 / USART2 / USART3，作为 LIN/UART 通道
  - SPI1，用于外部 Flash/TFT 相关外设
  - I2C，用于 OLED 或扩展器件
  - ADC1 + DMA，用于多通道模拟量采集
  - TIM1/2/3/4/6/7/14/15/16/17，用于 PWM、周期任务和时间基准

### 3.2 CAN 架构

#### CAN1

CAN1 是主要业务控制总线，承担：

- 接收整车/上位机控制命令。
- 接收 RC 控制报文：
  - `0x064`：毫米/位置控制。
  - `0x065`：百分比控制。
- 接收环境/门控报文，例如 `0x052`。
- 接收 LIN 桥接数据，例如 `0x104`、`0x105`。
- 发送 ACK：
  - 通用 ACK。
  - Action Done。
  - XY Done。
  - XYMove Done。
  - Door/Relay ACK。

#### CAN2

CAN2 作为扩展总线，配合电机控制、透传或其他 CAN 节点使用。实际用途需结合接线和上位机协议确认。

### 3.3 LIN 架构

板卡配置三路 LIN/UART 通道：

- LIN1：USART1
- LIN2：USART2
- LIN3：USART3

固件中对 LIN 接收采用中断方式，启动阶段使能 LIN 收发器 `LINx_EN`。LIN 数据处理包含：

- 读取接收字节/帧。
- 更新调试变量。
- 识别常见 RID：`0x22`、`0x34`、`0x35`、`0x36`。
- 未知 RID 进入通用处理流程。

### 3.4 电机与运动机构

当前固件定义的电机 ID 如下：

| 电机 | CAN ID | 作用 |
| --- | --- | --- |
| M1 | `0x0D1` | Y 轴相关，D1/D2 |
| M2 | `0x0D2` | Y 轴相关，D1/D2 |
| M3 | `0x0D3` | X 轴相关，D3/D4 |
| M4 | `0x0D5` | Z 轴/预留，D5 |

运动控制核心思路：

- 上位机发目标 XY。
- 固件根据屏幕标定参数转换成机械目标坐标。
- 调用位置控制循环发送目标给电机。
- 等待当前位置进入容差范围。
- 到位后执行 Z_code、XYMove 或 Reset 等后续动作。
- 最后发送完成 ACK 并释放锁定状态。

### 3.5 继电器、KL15 与门控输出

板卡包含多路继电器输出，固件中用于：

- KL15 通道控制。
- 左前门、右前门、左后门、前引擎盖、后备箱、预留门控输出。
- LIN 继电器/通道切换。
- 其他环境/车辆测试信号模拟。

门控逻辑会检测门状态是否变化，只有变化时才驱动 GPIO 并发送门控 ACK，减少重复输出和总线报文噪声。

### 3.6 OLED、按键与拨码

- OLED 用于显示版本、ID 模式、运行状态、错误信息、LIN/CAN 调试信息。
- 按键用于现场手动移动、标定和确认：
  - `UP/DOWN`：控制 Y 轴。
  - `LEFT/RIGHT`：控制 X 轴。
  - `BUTTON`：确认或触发。
- 拨码用于选择工作模式和坐标解释方式。

### 3.7 Flash 与参数保存

外部 Flash 主要用于保存：

- 屏幕标定坐标：`DispX0`、`DispY0`、`DispX1`、`DispY1`。
- UID/密钥相关参数。
- 其他系统配置。

启动时可通过 `CFG_BOOT_FLASH_VERIFY_RUN` 控制是否进行 Flash 验证。

## 4. 关键电路设计逻辑

> 本节从固件控制逻辑反推关键电路用途，用于理解硬件设计目的。具体器件型号、电平范围、保护参数应以原理图和数据手册为准。

### 4.1 电源与 KL15 控制逻辑

KL15 通常代表点火/附件电源状态模拟。板卡通过 MCU GPIO 控制 KL15 继电器：

- 当需要模拟触笔按压或车辆上电动作时，固件可拉高 KL15 继电器输出。
- 动作保持一段时间后释放。
- 在异常保护或动作完成后，固件会清除相关状态，避免继电器长时间误保持。

设计要点：

- KL15 输出建议使用继电器或低边/高边驱动隔离 MCU 与车端负载。
- 继电器线圈应配置续流保护。
- 若外部负载为感性或车载电源环境，应考虑 TVS、防反接和浪涌保护。

### 4.2 CAN 接口设计逻辑

CAN1/CAN2 接口用于整车/上位机/电机控制通信。

设计要点：

- MCU FDCAN_TX/RX 应连接 CAN 收发器。
- CANH/CANL 需满足差分阻抗和终端匹配要求。
- 根据总线拓扑选择是否焊接 120Ω 终端电阻。
- 车载环境建议增加 TVS、共模电感或 ESD 保护。
- CAN1 偏业务控制，CAN2 可用于扩展或电机网络，布线时应区分接口标识。

### 4.3 LIN 接口设计逻辑

LIN1/2/3 通过 UART 配合 LIN 收发器实现。

设计要点：

- MCU TX/RX 连接到 LIN 收发器逻辑侧。
- LIN 总线侧连接单线 LIN Bus。
- `LINx_EN` 用于使能收发器。
- 若使用 `NRES`，应保证上电后释放复位。
- LIN 总线需要合适的上拉和保护设计，具体按 LIN 收发器推荐电路实现。

### 4.4 电机控制接口逻辑

固件以 CAN ID 区分不同电机：M1/M2/M3/M4。设计上可采用 CAN 电机驱动器或电机控制板。

设计要点：

- Y 轴双电机需要保证方向、比例、零点一致。
- X 轴与 Y 轴的机械坐标方向需通过标定和软件映射统一。
- 电机驱动供电应与 MCU 逻辑电源隔离/滤波，避免电机启动电流影响 MCU。
- 建议预留急停、限位或故障反馈接口，软件保护不能完全替代硬件保护。

### 4.5 ADC 与光照传感器逻辑

ADC1 配置 6 路采样，通过 DMA 获取数据。光照传感器逻辑将 12bit ADC 值映射为 0~255 等级，并根据阈值生成报警等级。

设计要点：

- 模拟输入应做 RC 滤波。
- ADC 输入电压不得超过 MCU 允许范围。
- 光照传感器若存在正/反向差异，可通过软件宏进行反转。
- 传感器地线与功率地应注意回流路径，避免电机噪声干扰。

### 4.6 SPI Flash 逻辑

外部 Flash 用于参数持久化。

设计要点：

- SPI 片选需要和其他 SPI 设备互斥。
- 写 Flash 前必须写使能，写后等待忙状态结束。
- 标定参数写入后建议读回校验。
- 启动自检可验证 Flash 读写可靠性；现场调试可临时跳过以缩短启动时间。

### 4.7 OLED 与调试显示逻辑

OLED 通过 I2C 或项目封装接口控制，用于现场无需电脑即可判断系统状态。

设计要点：

- OLED 显示内容应避免过高刷新频率，减少 I2C 占用。
- 错误状态应优先显示，例如电机卡滞、超时、系统错误。
- 对于现场交接，建议统一显示行含义，例如版本/模式/运行状态/调试计数。

## 5. 使用流程

### 5.1 上电前检查

1. 确认板卡供电电压和极性正确。
2. 确认 CAN、LIN、继电器、电机驱动连接正确。
3. 确认电机机械结构无卡阻，触笔/执行机构处于安全位置。
4. 确认急停或断电手段可用。
5. 检查拨码 `IO_CFG_1~4` 是否符合本次测试模式。
6. 如需跳过 Flash 验证，将 `CFG_BOOT_FLASH_VERIFY_RUN` 设置为 `0` 并重新编译烧录。

### 5.2 上电启动流程

1. MCU 初始化 HAL、时钟、GPIO、DMA、ADC、CAN、LIN、SPI、TIM、I2C/OLED。
2. OLED 显示启动信息和版本号。
3. 读取拨码配置，确定运行模式。
4. 根据 `CFG_BOOT_FLASH_VERIFY_RUN` 选择：
   - `1`：执行 Flash 自检和 UID/密钥读取。
   - `0`：跳过 Flash 自检，使用快速启动流程。
5. 启动 ADC DMA、PWM、CAN、LIN 接收。
6. 初始化电机保护状态。
7. 进入主循环，周期执行门控、通信处理、运动控制、OLED 更新等逻辑。

### 5.3 屏幕坐标标定流程

1. 上电时根据提示进入标定流程。
2. OLED 提示设置 `X0/Y0`。
3. 使用方向键移动触点到屏幕零点：
   - `UP/DOWN` 调整 Y。
   - `LEFT/RIGHT` 调整 X。
4. 按下确认键保存 `X0/Y0` 到 Flash。
5. OLED 提示设置 `X1/Y1`。
6. 使用方向键移动触点到屏幕另一角。
7. 按下确认键保存 `X1/Y1` 到 Flash。
8. 固件读回 Flash 验证写入值，若失败则提示错误并重试。

### 5.4 CAN 控制运动流程

#### 5.4.1 `0x064` 毫米/位置控制

1. 上位机发送 `0x064` 报文。
2. 固件解析：
   - `buf[0..1]`：X 目标毫米值。
   - `buf[3..4]`：Y 目标毫米值。
   - `buf[2]`：X_Mov。
   - `buf[5]`：Y_Mov。
   - `buf[6]`：Z 值。
   - `buf[7]`：低 2 bit 为 `Z_code`，高位包含 Reset 信息。
3. 若不是 Reset，目标坐标按 `屏幕零点 + 输入值` 转换。
4. 检查目标是否在标定范围内。
5. 执行 XY 移动。
6. XY 到位后发送 XY Done ACK。
7. 若存在 `Z_code`，执行触笔/KL15 保持动作。
8. 若存在 `XYMove`，继续执行二段移动并发送 XYMove Done ACK。
9. 若存在 Reset 且当前序列允许，最后执行回零。
10. 清除本次命令状态并释放锁。

#### 5.4.2 `0x065` 百分比控制

1. 上位机发送 `0x065` 报文。
2. 固件读取 `IO_CFG_4`：
   - `id4 = 0`：按 `/100` 百分比映射。
   - `id4 = 1`：按 `/255` 兼容旧版映射。
3. 将百分比映射到 `X0~X1`、`Y0~Y1` 标定区间。
4. 后续执行流程与 `0x064` 类似。

### 5.5 手动按键移动流程

1. 确认系统未被远程命令锁定，即 `TA531_Lock == 0`。
2. 按键移动：
   - `UP`：Y 正方向。
   - `DOWN`：Y 负方向。
   - `LEFT`：X 负方向。
   - `RIGHT`：X 正方向。
3. 固件根据按键是短按还是持续按下选择不同步长。
4. 每次移动前调用坐标限制逻辑，避免越界。
5. OLED 显示当前方向，例如 `Go X+`、`Go Y-`。

### 5.6 门控/继电器使用流程

1. 上位机通过 CAN 报文更新门控状态。
2. 固件比较当前状态与上次状态。
3. 若状态变化，驱动对应继电器 GPIO：
   - 左前门
   - 右前门
   - 左后门
   - 前引擎盖
   - 后备箱
   - 预留通道
4. 打包门控状态，通过 CAN1 发送 ACK。

### 5.7 LIN 使用流程

1. 上电后固件启动 USART1/2/3 的 LIN 接收。
2. 收到 LIN 数据后进入中断/处理流程。
3. 识别常见 RID：`0x22`、`0x34`、`0x35`、`0x36`。
4. 更新内部状态、调试计数和 OLED 显示。
5. 需要转发或上报时通过 CAN ACK/状态报文输出。

### 5.8 异常处理流程

#### 电机保护触发

触发条件包括：

- X/Y 方向反复变化超过阈值。
- 位置长期无有效变化，疑似卡滞。
- 运动超过最大允许时间。

触发后流程：

1. 设置保护标志和错误类型。
2. OLED 显示错误信息。
3. 蜂鸣提示。
4. 停止当前动作并尝试回安全位置。
5. 清除 Z_code、XYMove、Reset 等命令残留。
6. 释放 `TA531_Lock`，系统回到可接收命令状态。

#### 越界命令

1. 固件收到目标位置后进行边界检查。
2. 若超出 `X0~X1` 或 `Y0~Y1` 范围：
   - 不执行移动。
   - 蜂鸣提示。
   - 将状态置为无任务/错误分支。
3. Reset 到机械零点属于特殊命令，可在允许条件下绕过屏幕边界限制。

## 6. 调试与维护建议

### 6.1 常用检查点

- OLED 是否显示版本和模式。
- CAN 是否能收到 `0x064/0x065` 并返回 ACK。
- LIN 收包计数是否变化。
- Flash 标定值是否正确读回。
- 手动方向键是否符合 `UP/DOWN=Y`、`LEFT/RIGHT=X`。
- 电机实际方向是否与 OLED 提示一致。
- KL15 和门控继电器是否按预期吸合/释放。

### 6.2 常见问题定位

| 现象 | 优先检查 |
| --- | --- |
| 坐标到不了中点 | 检查屏幕零点偏移、标定值、坐标换算方向 |
| UP/DOWN 动 X | 检查按键映射和 CAN 打包/解包轴顺序 |
| 第二条同 XY 命令不执行 Z | 检查 `Z_code`、`TA531_RC1_fg`、`TA531_Lock`、XY 到位判断 |
| 电机卡住后无响应 | 检查运动保护错误类型、是否已释放 `TA531_Lock` |
| Flash 启动慢 | 调试阶段可将 `CFG_BOOT_FLASH_VERIFY_RUN` 改为 `0` |
| LIN 无数据 | 检查 LIN_EN、收发器供电、波特率、RID 是否匹配 |
| 继电器不动作 | 检查门状态变量、GPIO、电源和驱动管/继电器线圈 |
